# 商业化架构改造任务清单

## 1. SQLite 不适合多用户并发

当前状态：

- 开发环境仍保留 SQLite，便于本地演示和快速验证。
- 已新增 `DATABASE_URL` 配置项。
- Docker Compose 已加入 PostgreSQL 服务。

已完成：

- 增加生产数据库配置入口。
- 增加 PostgreSQL 容器基础配置。

下一步：

- 将当前 sqlite3 直连代码迁移为 SQLAlchemy。
- 支持 `postgresql://` 连接。
- 增加 Alembic 数据库迁移。

## 2. ASR 和 API 在同一进程

当前状态：

- 已新增异步转写任务接口。
- 已新增线程池任务执行器。

已完成接口：

```http
POST /api/audio/{audio_id}/transcribe-jobs
GET  /api/jobs/{job_id}
WS   /ws/jobs/{job_id}
```

下一步：

- 将线程池替换为 Redis + Celery / RQ。
- 独立部署 ASR Worker。
- API 只负责任务创建和状态查询。

## 3. 缺少登录权限

已完成：

- 新增登录接口。
- 新增 Bearer Token 会话。
- 核心业务 API 已加入登录保护。
- 前端新增登录栏。

接口：

```http
POST /api/auth/login
GET  /api/me
```

默认开发账号：

```text
admin / admin123
```

生产要求：

- 密码哈希存储。
- 用户表和角色表。
- RBAC 权限控制。
- Token 过期和刷新。

## 4. 缺少完整审计页面

已完成：

- 审计日志表增加 `user_id` 和 `detail_json`。
- 核心操作写入审计日志。
- 新增审计查询接口。
- 前端新增“查看审计”入口。

接口：

```http
GET /api/audit-logs
```

已审计操作：

- 登录
- 上传音频
- 创建转写任务
- 转写音频
- 生成病历
- 确认病历
- 导出病历
- 文件清理
- EMR 同步骨架调用

## 5. 缺少任务队列

已完成：

- 新增内存任务队列和线程池执行器。
- 支持异步转写任务。
- 支持任务状态查询。

当前限制：

- 服务重启后任务状态会丢失。
- 多实例部署无法共享任务。

生产建议：

- Redis + Celery。
- 任务状态持久化到 PostgreSQL。
- WebSocket 推送任务进度。

## 6. 缺少 WebSocket

已完成：

- 新增任务状态 WebSocket。
- 新增实时音频 WebSocket 转写骨架。

接口：

```text
WS /ws/jobs/{job_id}
WS /ws/audio/transcribe
```

下一步：

- 前端实时录音从 HTTP 分段上传切换为 WebSocket 音频流。
- 支持增量转写和最终转写修正。

## 7. 缺少文件生命周期管理

已完成：

- 新增文件清理接口。
- 支持清理实时 chunk 和 WebSocket 临时音频。

接口：

```http
POST /api/files/cleanup?max_age_hours=24
```

下一步：

- 增加定时任务。
- 增加文件保留策略。
- 增加音频加密存储。
- 增加文件访问权限控制。

## 8. 缺少 HIS / EMR 集成

已完成：

- 新增 EMR 同步骨架接口。
- 新增集成日志表。

接口：

```http
POST /api/integrations/emr/sync
```

当前返回：

```json
{
  "status": "pending_external_integration",
  "message": "HIS/EMR 对接骨架已创建，需配置医院接口后启用真实同步。"
}
```

下一步：

- 根据医院接口文档实现真实同步。
- 支持 REST / HL7 / FHIR / 数据库视图。
- 支持同步失败重试。
- 支持同步状态页面。
